# 原始 PCB/原理图整理：湖师·N20 循迹小车 V0.1.x

> 来源：用户上传的 `焊接图.pdf` 与 `原理图.pdf`。本文件把图纸信息转成工程可读说明，后续 AI/同学接手时先看这里，再看 `pinmap.md` 和 `wiring-guide.md`。

## 1. 板卡定位

板名：**湖师·N20循迹小车 V0.1.0 / 智能小车扩展板 V0.1.2**。

该 PCB 是给 MSPM0G3507 开发板/核心板使用的小车扩展板，核心能力包括：

1. 电源输入与 5V 降压；
2. 四路 N20 电机驱动；
3. 电池电压 ADC 采集；
4. 五路红外/灰度循迹接口；
5. 超声波避障接口；
6. 蓝牙串口模块接口；
7. 2.4G/nRF24L01 遥控器接口；
8. 摄像头/视觉模块扩展口；
9. I2C、UART、IO、SWD/SWC、XJ01~XJ08 扩展口；
10. START/MODE 两个按键；
11. 蜂鸣器；
12. 左/中/右状态灯或车灯。

## 2. 与之前采购方案的差异

之前通用方案建议外接 TB6612FNG。根据本 PCB 原理图，本扩展板已经集成了 4 路 `RZ7899` 电机驱动芯片。

因此：

- 使用这块 PCB 时，第一版可以 **不额外接 TB6612**；
- 电机控制以板载 `RZ7899` 为主；
- 每一路电机驱动只需要两个 MCU 控制信号：`+` 与 `-`；
- 若后续发现板载驱动电流不足或接口不适合，再外接 TB6612 作为升级方案。

## 3. 电源结构

原理图中的电源路径：

```text
电池座/P3 输入 BAT
→ D1 SS34 防反接/保护
→ SW1 总开关
→ 7.4V 电源母线
→ U6 降压模块输出 5V
```

板上电源节点：

| 电源 | 用途 |
|---|---|
| `7.4V` | 电池输入、电机驱动 VM/VCC |
| `5V` | 超声波、扩展口、部分外设供电 |
| `3V3` | MSPM0 逻辑、蓝牙 KEY、nRF24L01、部分信号上拉 |
| `GND` | 全板公共地 |

注意：电机电流较大，第一次上电必须架空车轮，并确认没有短路。

## 4. 电池电压采集

原理图中 ADC 采集采用三只 10k 电阻分压：

```text
7.4V → R4 10k → R5 10k → ADC 节点 → R6 10k → GND
```

因此：

```text
ADC 节点电压 ≈ 电池电压 / 3
电池电压 ≈ ADC 电压 × 3
```

示例：

```text
电池 8.4V 满电时，ADC 约 2.8V
电池 7.4V 标称时，ADC 约 2.47V
电池 6.6V 较低时，ADC 约 2.2V
```

该分压比适合 3.3V ADC 输入范围。

## 5. 电机驱动结构

板载 4 个 RZ7899：

| 驱动芯片 | 电机代号 | 方向/控制输入 | 输出网络 |
|---|---|---|---|
| U4 | `LQ` | `LQ-`、`LQ+` | `BO79`、`FO79` |
| U7 | `LH` | `LH-`、`LH+` | `BO1012`、`FO1012` |
| U3 | `RQ` | `RQ-`、`RQ+` | `BO46`、`FO46` |
| U2 | `RH` | `RH-`、`RH+` | `BO13`、`FO13` |

建议命名：

```text
LQ = Left-Qian  = 左前轮
LH = Left-Hou   = 左后轮
RQ = Right-Qian = 右前轮
RH = Right-Hou  = 右后轮
```

如果你只做两轮差速 + 万向轮，可以优先只接：

```text
LQ + RQ
```

或者根据实际机械位置选择：

```text
LH + RH
```

不要四个电机一开始全上，先用两路把控制跑稳。

## 6. 按键与指示灯

| 名称 | 网络 |
|---|---|
| START 按键 | `KEY_S` |
| MODE 按键 | `KEY_M` |
| 左车灯 | `LED_L` |
| 模式/中间指示灯 | `LED_M` |
| 状态/右车灯 | `LED_S` / `LED_R`，实际用途按丝印和代码定义 |
| 蜂鸣器 | `BUZZER` |

按键建议软件上拉/下拉状态以实际原理图和实测为准。第一次写代码时必须串口打印按键原始值。

## 7. 通信/扩展接口

| 模块 | 接口 |
|---|---|
| 蓝牙模块 U5 | `EN/KEY`、`VCC`、`GND`、`MCU_TXD`、`MCU_RXD`、`STATE` |
| 超声波 H1 | `5V`、`TRIQ`、`ECHO`、`GND` |
| 2.4G H4 | `3V3`、`GND`、`CSN`、`CE`、`MOSI`、`MISO`、`SCK`、`IRQ` |
| 摄像头 H3 | `PC8/PC9/PC11/...` 多路 GPIO + `5V/GND` |
| 顶部扩展 H2 | I2C、UART、SWD/SWC、5V、3V3、XJ01~XJ08 |

## 8. 重要风险点

1. `TRIQ` 是图纸/丝印里的写法，实际功能应理解为超声波 Trigger。
2. 电机驱动是板载 RZ7899，不是 TB6612；代码不能直接套 TB6612 的 AIN1/AIN2/PWMA 模型。
3. 电机控制口没有单独 EN/PWM 引脚，应采用两输入 H 桥控制：一脚 PWM、一脚低电平；反转时互换。
4. nRF24L01 使用 3.3V，不能接 5V。
5. 超声波 VCC 是 5V，但 ECHO 是否需要分压要看所用模块输出电平；若 ECHO 为 5V，接 MSPM0 前建议加分压/电平转换。
6. ADC 采电池电压已分压，代码里要乘以 3。
7. 第一次上电前用万用表测 7.4V、5V、3V3、GND 是否正常。
