# 接线教程：MSPM0G3507 + 湖师·N20 循迹小车 PCB

> 本教程按“先电源、再主控、再串口/OLED、再电机、再循迹”的顺序整理。第一次测试时，小车轮子不要落地，先架空验证方向和 PWM。

## 1. 接线总原则

这块 PCB 已经集成电池输入、5V 降压、四路 RZ7899 电机驱动、按键、蜂鸣器、车灯、蓝牙、超声波、2.4G、摄像头和顶部扩展口。

第一版不要一次性把所有模块都接上。推荐最小系统：

```text
电池输入
MSPM0G3507 开发板
H2 串口 TXD/RXD/GND
H2 OLED SCL/SDA/VCC/GND
LQ 左轮电机
RQ 右轮电机
XJ01~XJ08 循迹传感器
```

## 2. 第一步：电源检查

打开主开关前，先用万用表检查：

| 测点 | 正常情况 |
|---|---|
| 7.4V 与 GND | 不短路 |
| 5V 与 GND | 不短路 |
| 3V3 与 GND | 不短路 |
| 电池正负极 | 极性正确 |

打开开关后，测量：

| 测点 | 预期值 |
|---|---|
| 7.4V 到 GND | 约 6.6V~8.4V |
| 5V 到 GND | 约 5.0V |
| 3V3 到 GND | 约 3.3V |
| ADC 到 GND | 约电池电压 / 3 |

如果 5V 或 3V3 不正常，先不要插开发板，也不要接电机。

## 3. 第二步：插 MSPM0G3507 开发板

按焊接图把开发板插到 PCB 中央 P1/P2 两排排母上：

- P1 对 P1；
- P2 对 P2；
- USB/Type-C 朝向与焊接图一致；
- 插入前确认没有错位一排或反插。

注意：MSPM0G3507 不使用 ST-LINK 下载。

## 4. 第三步：接 USB 串口

优先使用 H2 顶部扩展口的 UART：

| USB 转串口 | PCB H2 |
|---|---|
| GND | GND |
| TXD | RXD |
| RXD | TXD |

串口参数建议：

```text
115200 8N1
```

TX/RX 要交叉，GND 必须相连。

## 5. 第四步：接 OLED

I2C OLED 接 H2：

| OLED | PCB H2 |
|---|---|
| VCC | 3V3 或 5V，按 OLED 模块要求 |
| GND | GND |
| SCL | SCL |
| SDA | SDA |

第一版 OLED 只显示基础信息：

```text
MODE: IDLE
BAT : 7.40V
LSPD: 0
RSPD: 0
```

## 6. 第五步：接两个 N20 电机

第一版只做两轮差速，建议：

| 逻辑轮 | 推荐电机接口 | 控制网络 |
|---|---|---|
| 左轮 | LQ | LQ+、LQ- |
| 右轮 | RQ | RQ+、RQ- |

若机械安装更适合后轮驱动，可改用：

| 逻辑轮 | 电机接口 | 控制网络 |
|---|---|---|
| 左轮 | LH | LH+、LH- |
| 右轮 | RH | RH+、RH- |

RZ7899 双输入控制建议：

| 动作 | + 输入 | - 输入 |
|---|---|---|
| 停止 | 0 | 0 |
| 正转 | PWM | 0 |
| 反转 | 0 | PWM |

第一次测试时，PWM 从小值开始，分别测试左轮、右轮，并记录哪个方向是“前进”。

## 7. 第六步：接循迹传感器

焊接图上有“循迹”接口，原理图中也把 `XJ01`~`XJ08` 引到顶部 H2。

推荐接法：

| 循迹模块 | PCB |
|---|---|
| VCC | 3V3 或 5V，按模块要求 |
| GND | GND |
| OUT1~OUT8 | XJ01~XJ08 或板上循迹接口对应信号 |

先不要跑车，先用串口打印每一路黑白状态：

```text
SENSOR: 00111100 ERR=-1.0
```

## 8. 按键、蜂鸣器、车灯

| 功能 | 网络名 |
|---|---|
| START 按键 | KEY_S |
| MODE 按键 | KEY_M |
| 蜂鸣器 | BUZZER |
| 左车灯 | LED_L |
| 中间/模式灯 | LED_M |
| 右车灯 | LED_R / LED_S，按实测定义 |

建议功能：START 负责启动/停止，MODE 负责切换模式。

## 9. 超声波模块 H1

| 超声波模块 | PCB H1 |
|---|---|
| VCC | 5V |
| Trig | TRIQ |
| Echo | ECHO |
| GND | GND |

注意：如果所用超声波模块 Echo 输出为 5V，接入 MSPM0 前建议做电平转换或分压。

## 10. 蓝牙模块 U5

| 蓝牙模块 | PCB |
|---|---|
| KEY/EN | EN |
| VCC | VCC，按模块供电要求确认 |
| GND | GND |
| TXD | MCU_RXD |
| RXD | MCU_TXD |
| STATE | STATE |

建议先用 USB 串口调参，USB 串口稳定后再接蓝牙。

## 11. 2.4G 模块 H4

| nRF24L01 | PCB H4 |
|---|---|
| VCC | 3V3 |
| GND | GND |
| CSN | CSN |
| CE | CE |
| MOSI | MOSI |
| SCK | SCK |
| IRQ | IRQ |
| MISO | MISO |

nRF24L01 只能接 3.3V，不能接 5V。

## 12. 视觉扩展 H3

H3 引出多路 GPIO 和电源：

```text
PC8 PC9 PC11 PA8 PG9 PD3 PB7 PB6 PB8 PB9 PA4 PA6 PC7 PC6 5V GND
```

当前阶段不建议直接上视觉。先把循迹、电机、编码器、PID 做稳。

## 13. 推荐上电顺序

1. 不接电机，只测电源；
2. 插 MSPM0，确认串口日志；
3. 接 OLED，确认显示；
4. 接一个电机，架空测试；
5. 接第二个电机，架空测试；
6. 接循迹模块，手动扫黑线；
7. 低速落地测试。

## 14. 常见问题

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 没有 5V | 降压、电池、开关或焊点问题 | 断电检查电源链路 |
| 电机一转开发板复位 | 电源电流不足、干扰大、线松 | 加电容、固定线束、确认共地 |
| 左右轮方向相反 | 电机方向或软件方向定义相反 | 在软件里设置方向系数 |
| 串口乱码 | 波特率不对、TX/RX 接反、没共地 | 115200 8N1，TX/RX 交叉 |
| OLED 不亮 | 地址、供电或 I2C 接线问题 | 检查 0x3C/0x3D、SCL/SDA |
| 循迹全是 0 或全是 1 | 阈值、高度、供电或输出极性问题 | 单路测试黑白状态 |
