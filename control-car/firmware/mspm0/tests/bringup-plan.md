# 固件 Bring-up 测试计划

## 1. BOOT 测试

目标：确认工程能运行。

验收：

- 串口输出版本号；
- OLED 显示 IDLE；
- 两个按键能打印原始值；
- 电池电压能显示。

## 2. GPIO 测试

目标：确认 LED、按键、蜂鸣器网络无误。

记录：

```text
KEY_S 原始值：
KEY_M 原始值：
LED_L 是否正常：
LED_M 是否正常：
LED_R/LED_S 是否正常：
BUZZER 是否正常：
```

## 3. 电机通道测试

目标：确认 LQ/RQ/LH/RH 四个通道方向和响应。

记录：

```text
LQ 正向对应网络：
RQ 正向对应网络：
LH 正向对应网络：
RH 正向对应网络：
最小可转 PWM：
安全最大 PWM：
```

## 4. 循迹传感器测试

目标：确认 XJ01~XJ08 或板上循迹接口的顺序。

记录：

```text
传感器从左到右：
黑线输出电平：
白底输出电平：
传感器高度：
```

## 5. PID 初测

目标：先低速、短时间测试，不追求速度。

记录：

```text
基础速度：
速度 PID：
循迹 PID：
现象：
是否摆动：
是否丢线：
```
