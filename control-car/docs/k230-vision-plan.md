# 庐山派 K230 视觉方案：control-car 小车/云台扩展

> 结论：`control-car/` 的视觉主线统一切换为 **立创·庐山派 K230 / CanMV K230**。K230 只做视觉协处理器，MSPM0G3507 继续做运动控制、PID、编码器、电机、OLED、按键和安全状态机。

---

## 1. 方案定位

K230 不替代 MSPM0G3507 主控，而是作为“摄像头 + AI/图像处理协处理器”。

```text
庐山派 K230 + 摄像头
  ├── 黑线/边线/色块/二维码/AprilTag/目标检测
  ├── 输出视觉偏差、目标坐标、识别状态
  └── UART 串口
        ↓
MSPM0G3507 主控
  ├── TB6612 电机控制
  ├── 编码器测速
  ├── 速度 PID + 转向 PID
  ├── 云台/舵机/步进电机控制
  └── OLED/蜂鸣器/按键/安全状态机
```

核心原则：**K230 负责“看见”，MSPM0 负责“动起来并稳定控制”。**

---

## 2. 为什么改用 K230

| 维度 | K230 作为主线的意义 |
|---|---|
| 已有硬件 | 用户已有庐山派 K230 模块，减少重复采购。 |
| 电赛扩展性 | 可覆盖视觉循迹、靶面识别、色块识别、二维码、AprilTag、YOLO 小模型。 |
| 和小车架构匹配 | K230 通过 UART 只输出结构化结果，不干扰 MSPM0 的实时控制。 |
| 后续上限 | 比普通灰度/简单视觉模块更适合复杂目标检测和加分项。 |

注意：第一阶段仍然不依赖 K230。只有当 **电机、编码器、灰度循迹、PID、UART 调参** 稳定后，才进入 K230 视觉扩展。

---

## 3. 必要配件

### 3.1 必需

| 器材 | 建议规格 | 用途 |
|---|---|---|
| microSD / TF 卡 | 8GB 或 16GB，Class 10 / U1 及以上 | K230 启动系统、保存脚本/模型。 |
| TF 读卡器 | USB-A 或 Type-C | 给 TF 卡烧录镜像。 |
| USB-C 数据线 | 支持数据传输 | K230 供电、串口/IDE 连接。 |
| 摄像头支架 | 可调俯仰角 | 固定视角，减少识别漂移。 |
| 5V 稳压模块 | 2A 以上，建议独立一路 | 给 K230 稳定供电。 |
| 杜邦线 / GH1.25 转杜邦线 | 视 K230 接口而定 | 连接 K230 UART 与 MSPM0。 |

### 3.2 推荐

| 器材 | 用途 |
|---|---|
| 补光 LED | 提高黑线、色块、靶面识别稳定性。 |
| 小型万向支架 / 3D 打印支架 | 让摄像头高度、角度可重复。 |
| 逻辑分析仪 | 排查 UART TX/RX、波特率、乱码问题。 |
| 备用 TF 卡 | 只作为备份，不在正式封箱后随意更换。 |

### 3.3 TF 卡速率要求

最低按 **Class 10 / U1** 选购，等价于最低持续写入约 **10MB/s**。不需要专门购买高价 U3/V30 卡；预算允许时买 16GB/32GB U3/V30 更稳，但不是必要条件。

## 4. 固件与系统

使用庐山派 K230 对应的 **CanMV MicroPython 固件/镜像**。

禁止混用：

- 不要把 Lite-K230D 固件烧到庐山派 K230；
- 不要把普通 Linux 镜像当作 CanMV 视觉训练主线；
- 不要现场依赖电脑、手机、网络或云端服务完成识别。

推荐流程：

```text
下载 K230 CanMV 镜像
→ 烧录到 TF 卡
→ 插入 K230
→ USB-C 上电
→ CanMV IDE 连接
→ 跑摄像头预览
→ 跑颜色/二维码/AprilTag/循迹例程
→ 通过 UART 输出结果给 MSPM0
```

---

## 5. UART 连接规范

### 5.1 物理连接

推荐使用 K230 的 UART2 作为对外通信口，波特率从 **115200 8N1** 起步。

```text
K230 TXD  → MSPM0 UART RX
K230 RXD  → MSPM0 UART TX
K230 GND  → MSPM0 GND
K230 5V   → 独立 5V 稳压输出
```

必须共地，否则串口通信会不稳定。

### 5.2 电平注意

- MSPM0G3507 逻辑电平为 3.3V；
- K230 串口通信也按 3.3V 逻辑对待；
- 不要把 5V TTL 串口直接接到 3.3V IO。

---

## 6. 通信协议

K230 到 MSPM0 使用“一行一帧”的 ASCII 协议，便于串口助手调试。

### 6.1 基础帧

```text
VISION,MODE=LINE,OK=1,CX=154,CY=102,ERR=-6,CONF=83
VISION,MODE=BLOB,OK=1,CX=121,CY=88,W=34,H=29,ID=RED,CONF=90
VISION,MODE=TAG,OK=1,ID=3,CX=118,CY=92,ROT=-12,CONF=95
VISION,MODE=QR,OK=1,DATA=START_A,CONF=99
VISION,MODE=NONE,OK=0,ERR=LOST,CONF=0
```

### 6.2 MSPM0 侧只关心的最小字段

| 字段 | 含义 |
|---|---|
| `MODE` | 当前视觉模式：`LINE` / `BLOB` / `TAG` / `QR` / `YOLO`。 |
| `OK` | 是否识别成功。 |
| `ERR` | 横向偏差，左负右正。 |
| `CX`,`CY` | 目标中心点坐标。 |
| `ID` | 标签/颜色/类别。 |
| `CONF` | 置信度或稳定度，0~100。 |

### 6.3 控制闭环建议

```text
K230 视觉帧率：10~30fps
MSPM0 控制周期：10ms
MSPM0 视觉数据超时：100~300ms
```

如果超过超时时间没有收到有效视觉帧，MSPM0 必须进入降速、搜索或停车状态，不能继续盲目高速运行。

## 7. 分阶段练习路线

### 阶段 K0：K230 单板点亮

目标：确认 K230 能启动 CanMV。

验收：

- TF 卡镜像烧录成功；
- K230 能启动；
- CanMV IDE 能连接；
- 摄像头预览正常。

### 阶段 K1：基础视觉例程

目标：跑通不用训练的视觉任务。

任务：

1. 摄像头预览；
2. 色块识别；
3. 二维码识别；
4. AprilTag 识别；
5. 简单找线/阈值分割。

### 阶段 K2：K230 → MSPM0 UART

目标：K230 发送坐标，MSPM0 正确解析。

验收：

- MSPM0 串口能收到 `VISION,...`；
- OLED 能显示 `MODE/OK/ERR/CX/CY`；
- 断开 K230 时 MSPM0 能超时报警。

### 阶段 K3：视觉辅助循迹

目标：灰度循迹是主线，视觉作为增强输入。

策略：

```text
低速/直线：灰度传感器为主
复杂区域/靶面/标志识别：K230 视觉辅助
视觉丢失：回退灰度循迹或停车
```

### 阶段 K4：视觉云台/靶面识别

目标：K230 输出靶心或目标中心，MSPM0 控制二维云台对准。

数据流：

```text
K230 检测目标中心 (cx, cy)
→ 计算图像中心偏差 err_x, err_y
→ UART 发给 MSPM0
→ MSPM0 云台 PID / 步进电机角度控制
```

### 阶段 K5：YOLO / kmodel 进阶

只在普通阈值、形状、AprilTag 不够时启用。

任务：

1. PC 端标注数据；
2. 训练轻量模型；
3. 转换为 K230 可运行模型；
4. 在 K230 端侧推理；
5. 仍然只通过 UART 给 MSPM0 发结果。

## 8. 电源与结构要求

K230 不要和电机直接共用一条细线供电。

推荐：

```text
2S 电池
├── TB6612 VM → 电机
├── 5V/3A 降压 → K230
└── 3.3V/5V 逻辑电源 → MSPM0、OLED、传感器

所有 GND 共地
```

结构上要保证：

- 摄像头支架牢固；
- 视角可重复；
- 线束固定，避免跑车时扯掉；
- K230 与电机线保持距离，减少干扰；
- TF 卡插好后不要在运行中拔插。

---

## 9. 与比赛合规的执行边界

通用原则：K230 和 TF 卡必须作为作品的一部分离线运行。正式封箱后，不应靠外部电脑、手机、网络服务器、云端 AI 或临时更换 TF 卡完成识别。

赛题如果明确禁止摄像头、视觉模块、外部处理器或特定通信方式，则以赛题要求为最高优先级。
