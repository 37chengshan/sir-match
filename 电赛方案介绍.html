为什么必须有DAC

核心
波形复刻（40分）需要输出任意补偿波形。DAC直接输出，精度可控；无DAC只能用AD9833逐谐波合成，复杂且易出错
SRAM
192KB SRAM可容纳1024点FFT缓冲区，实现完整的谐波分析和补偿
生态
F407是电赛/嵌入式竞赛最常用MCU，社区资源极丰富，CubeMX完美支持

核心算法

Goertzel 幅度相位提取

O(N)计算单频率DFT，同时输出幅度和相位。比完整FFT快10×以上，完美匹配扫频应用。

增量式PID + 积分限幅 + 输出限幅。赛前MATLAB预设参数，上电自动阶跃微调。

控制精度：全频段(100Hz~3kHz)输出误差 <5%

RLC解析拟合 + 逆滤波波形复刻

从频率响应提取f₀和Q值 → 解析计算L和C → 枚举验证R → 设计逆滤波器H⁻¹(s)。对矩形波需做FFT谐波分解后分别补偿各次谐波。

三审核改进

P0
.c源文件全部缺失 — STM32项目仅9个头文件，0个实现文件。预计需编写~1,880行C代码，必须立即开始。
P0
幅度控制方案待确定 — 推荐MCP41010数字电位器(SPI, 256级) + OPA2134可编程增益作为AD9833后级幅度调节。
P1
矩形波复刻需FFT — 当前Goertzel方案只能做单频，矩形波需要谐波分解。添加FFT路径用于谐波分析。
P2
面包板高频布局 — 50kHz扫频时寄生电容影响增大。关键模拟路径远离数字信号线，ADC输入紧靠MCU引脚。
P2
F407时钟树配置 — 168MHz需正确配置PLL: HSE 8MHz → PLL×336 → /2 = 168MHz。注意APB1=42MHz(定时器84MHz)，APB2=84MHz(定时器168MHz)。
