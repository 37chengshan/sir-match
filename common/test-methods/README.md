# common/test-methods 通用测试方法

这里放跨题目复用的测试、标定、数据记录和误差分析方法。

## 1. 当前测试方法索引

| 文件 | 用途 |
|---|---|
| `scope-measurement.md` | 示波器测量 Vin/Vout、相位、Vpp |
| `csv-format.md` | 串口导出 CSV 的统一格式 |
| `calibration.md` | ADC 零点、幅度、相位校准流程 |
| `error-analysis.md` | 报告里的误差来源分析 |

## 2. 通用测试原则

1. 每个模块先单独测试，再集成；
2. 每次改硬件后先测电源；
3. 模拟信号先用示波器看，再相信 ADC 数据；
4. 串口输出必须包含时间、模式、频率、电压、错误码；
5. 关键测试必须保存 CSV 和示波器截图；
6. 报告中理论值、测量值、误差百分比必须对应。

## 3. 最小测试记录格式

```text
日期：
题目：
模块：
硬件版本：
固件版本/commit：
输入条件：
测量仪器：
测试结果：
问题：
下一步：
```

## 4. 适用题目

- `G-circuit-model/`：频响、Vpp、相位、波形复现；
- `control-car/`：电机 PWM、编码器速度、循迹偏差、PID 参数。
