# 电赛备赛总要求

## 1. 总目标

本仓库服务于电赛备赛，目标不是临时堆资料，而是形成可复用的工程积木：

- 主控平台积木；
- 电源积木；
- 信号产生积木；
- 数据采集积木；
- 显示交互积木；
- 通用算法积木；
- 测试与报告积木。

当前优先级最高的平台是 **TI MSPM0G3507**。

## 2. 目录总规则

每个题目必须独立建文件夹：

```text
problems/<题目编号-英文短名>/
```

每个题目文件夹至少包含：

```text
docs/          方案、评分拆解、采购清单、接线说明
firmware/      固件工程
simulation/    MATLAB/Python仿真
hardware/      原理图、PCB、接线图
test-data/     示波器截图、CSV、标定结果
report/        报告素材和最终报告
web/           展示页或可视化说明
```

通用内容统一放：

```text
common/algorithms/    Goertzel、PID、FFT、滤波识别、拟合等
common/hardware/      电源、显示、AD9833、ADC前端、DAC输出等
common/test-methods/  示波器测法、频响测试、误差分析等
```

## 3. 文档要求

每个题目的方案文档至少包括：

1. 题目理解；
2. 得分点拆解；
3. 总体方案；
4. 硬件框图；
5. 软件状态机；
6. 算法路线；
7. 采购清单；
8. 调试计划；
9. 风险点；
10. 最小可运行验收标准。

## 4. 代码要求

每个题目的固件工程应尽量按以下方式组织：

```text
firmware/<platform>/
├── Core/
│   ├── App/          应用状态机
│   ├── Drivers/      外设驱动
│   ├── Algorithms/   题目专用算法
│   └── Config/       引脚、频率、参数配置
├── Tools/            串口脚本、烧录脚本、数据转换脚本
└── README.md         编译、烧录、接线、运行说明
```

## 5. 采购要求

采购清单分三层：

- 第一批：能跑通最小闭环的必买器材；
- 第二批：提升稳定性和调试效率的增强器材；
- 第三批：冲高分、做成品和写报告用的器材。

采购清单必须写：

- 名称；
- 型号/规格；
- 数量；
- 用途；
- 必要性；
- 估价；
- 替代方案。

## 6. 调试要求

每个模块都要有独立验收标准：

- 主控能下载、串口能打印；
- 信号源能输出指定频率；
- ADC 能采到稳定波形；
- DAC 能输出查表波形；
- 显示能显示关键状态；
- 算法能用仿真数据和实测数据双验证。

## 7. 重要禁止项

- 不要把新题目资料散放在根目录。
- 不要在没有跑通硬件闭环前继续堆复杂算法。
- 不要用 ST-LINK 下载 MSPM0。
- 不要把 STM32 旧方案当作最终 TI 平台方案。
- 不要把通用算法复制到每个题目里，统一放 common。
